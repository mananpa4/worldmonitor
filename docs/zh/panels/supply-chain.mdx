**Supply Chain** 面板（内部 id `supply-chain`）是一切供应链相关内容的主要入口：综合航运压力分数、带价格变动的承运人和 ETF 行、矿物集中度视图，以及最重要的是 **[Scenario Engine](/zh/scenario-engine)** 触发按钮，允许 PRO 用户在地图和面板状态上激活预构建的中断场景。

## 面板显示内容

综合指标区域：

- **航运压力标题** — 一个综合压力分数（0-100），带着色仪表，按级别分桶（`critical` / `elevated` / `moderate` / `normal`）。
- **承运人/ETF/指数行** — 每代码价格、百分比变化（`▲`/`▼` 箭头，带看涨/看跌色调）、紧凑迷你图和类型徽章（`ETF` / `IDX` / `CARR`）。
- **关键矿物行** — 每矿物前 3 生产国集中度、HHI 和风险评级（`critical` / `high` / `moderate` / `low`）。
- **Scenario Engine 触发卡片** — 每场景触发按钮（`sc-scenario-trigger` / `sc-scenario-btn`），将模板 id 分派到场景 worker。免费用户看到按钮但在激活时遇到 PRO 门控。请参见 [Scenario Engine](/zh/scenario-engine) 了解完整工作流。
- **活动场景横幅**（当场景运行时）— 图标、场景名称、可选参数标签（持续时间 + 成本增加）、受影响最大的国家、关闭 ×。当模板参数被填充时，渲染额外的 "Simulating …" 标语。横幅构建位于 `src/components/SupplyChainPanel.ts` 的 `renderScenarioBanner()` 中。

面板 id 为 `supply-chain`；规范组件为 `src/components/SupplyChainPanel.ts`。按变体的标题："Supply Chain"（full/finance）或 "Supply Chain & Logistics"（commodity）。

## 如何访问

- **Cmd+K**：输入 *supply chain*、*shipping* 或 *logistics*。
- **按变体的可用性**：在 **full/geopolitical**（priority 1）、**finance**（priority 1）和 **commodity**（priority 1）变体中注册并默认启用。在 tech 或 happy 变体中不存在。来源：`FULL_PANELS`、`FINANCE_PANELS`、`COMMODITY_PANELS` 中的 `'supply-chain'` 条目。桌面端 full 变体注册额外叠加 `premium: 'enhanced'`（`src/config/panels.ts` 中的 `_desktop &&` 子句）。

## 数据来源

读取缓存的供应链数据集（综合压力、承运人列表、矿物、场景模板）。Scenario Engine 触发调用 `POST /api/scenario/v1/run-scenario` 并轮询 `GET /api/scenario/v1/get-scenario-status` — 请参见 [Scenarios API](/zh/api-scenarios) 了解 HTTP 合约。

## 刷新频率

面板在挂载和用户触发刷新时读取。Scenario Engine 作业是异步的；面板轮询作业状态端点，在 worker 写入 `status: "done"` 后绘制横幅。

## 层级与门控

Supply Chain 面板本身在 Web 端**免费**（full / finance / commodity 变体）。桌面端 full 变体注册叠加 `premium: 'enhanced'`。**Scenario Engine 激活是 PRO**，无论 Web 还是桌面 — 免费用户看到触发按钮但在激活时遇到 `scenario-engine` 门控。
